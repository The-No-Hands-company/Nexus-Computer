# Nexus.computer Roadmap

Nexus is not trying to merely match Zo Computer. The goal is to become the preferred personal cloud AI computer for people who want more freedom, more control, and better results.

## Product principle

- Free as in freedom and free as in price
- Open source from the start
- Private by default
- Self-hostable and portable as a standalone instance or integrated into Nexus Cloud
- No ads, no subscriptions, no paywalls
- Built for real work, not platform lock-in
- Community-driven: if people ask for useful features, we build them
- Nexus AI is the default intelligence layer for Nexus Computer

## What Zo already does well

Based on Zo's public docs and repo, the benchmark includes:

- Multiple leading AI models
- File support across notes, code, docs, audio, video, images, PDFs, and ebooks
- Built-in transcription, image generation, and video generation
- Personal storage on the user's server
- Snapshots and restore
- Rules, personas, themes, and automated agents
- Hosting websites and services
- Text, email, and Telegram access
- Browser access
- Built-in integrations
- Custom integrations and MCP support
- File syncing from local devices
- A public community surface through docs, Discord, X, GitHub, and ambassadors

## Nexus parity goals

These are the features we should match quickly:

1. **AI workspace**
   - Chat with multiple models
   - Long-running agentic tasks
   - File-aware conversations
   - Streaming responses

2. **File system**
   - Create, edit, browse, search, and preview files
   - Safe persistent workspace
   - Upload/download support
   - Snapshots and restore

3. **Computer layer**
   - Terminal access
   - App and script execution
   - Package installation
   - Process management
   - Web hosting for user projects

4. **User control**
   - Rules
   - Personas
   - Model choice
   - Theme customization
   - Import/export of everything

5. **Reachability**
   - Web access first
   - Mobile-friendly experience
   - Optional email/text/other channels later

## Where Nexus should be better

These are the differentiators that should make Nexus the better choice:

### 1. Radical ownership

- Open-source core
- Easy self-hosting
- Simple backup and migration
- Data stored in open formats
- No vendor lock-in
- Clear export path for the full computer

### 2. Better privacy

- Minimal telemetry
- Private-by-default sharing
- Optional encryption for sensitive data
- Per-folder and per-project access boundaries
- Clear audit trail for AI actions

### 3. Better agent UX

- Faster command execution feedback
- Cleaner tool logs
- More reliable file edits
- Better recovery after failures
- Safer permissions for destructive actions
- Clear “what happened” summaries after each task

### 4. Better developer experience

- One-command setup
- Better templates
- Opinionated but simple architecture
- Built-in project scaffolding
- First-class Git workflows
- Better terminal and service management

### 5. Better community response

- Public feature request board
- Transparent roadmap
- Voting on priorities
- Fast turnaround on high-value requests
- Release notes that explain what changed and why

### 6. Better product philosophy

- No paywalls around basic utility
- No artificial limits used as bait
- No dark patterns
- No forced upsells
- No “premium” gating of essential workflows

## Community listening loop

We should treat user feedback as product input, not noise.

### Inputs we should watch

- Zo docs and public announcements
- GitHub repos and issues
- Discord feedback
- X posts mentioning Zo
- Support requests and feature asks
- Requests from our own Nexus users

### Process

1. Collect requests into a single backlog
2. Group them by theme
3. Tag each item as parity, improvement, or new territory
4. Prioritize by usefulness, frequency, and effort
5. Ship the smallest useful version first
6. Announce what was added and what still remains
